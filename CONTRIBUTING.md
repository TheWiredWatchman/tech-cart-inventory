# Contributing to Tech Cart Inventory System

Thank you for your interest in contributing! This project was built to help maintenance technicians save time, and your contributions can help even more teams.

## üõ†Ô∏è How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs
- Include browser version and operating system
- Provide steps to reproduce the issue
- Screenshots are helpful!

### Suggesting Features
- Open a GitHub Issue with the "enhancement" label
- Describe the use case and how it would help technicians
- Be specific about the desired functionality

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Keep the single-file architecture (everything in index.html)
   - Test in multiple browsers (Chrome, Firefox, Edge)
   - Ensure mobile responsiveness is maintained
4. **Commit with clear messages**
   ```bash
   git commit -m "Add: description of feature"
   ```
5. **Push and create a Pull Request**

## üìù Code Style Guidelines

### JavaScript
- Use clear, descriptive variable names
- Comment complex logic
- Keep functions focused and single-purpose
- Maintain the existing code organization structure

### CSS
- Use existing color variables where possible
- Maintain responsive design
- Test on mobile devices

### HTML
- Use semantic elements where appropriate
- Maintain accessibility (alt text, ARIA labels)
